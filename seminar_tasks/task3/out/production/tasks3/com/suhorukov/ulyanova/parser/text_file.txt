List, Set и Map
Основные примитивы:
1. Collection - упорядоченный список элементов, доступ только через итератор
2. List – упорядоченный список элементов с возможностью доступа по индексу
3. Set – множество без дубликатов и без доступа по индексу
4. Map – множество пар ключ-значение, где ключи не повторяются. Нет доступа по
индексу
Их основные реализации:
1. ArrayList — аналог массива переменного размера, очень быстрый доступ по индексу
2. LinkedList — список элементов связанных друг с другом ссылками на соседей. Очень
медленный доступ по индексу, быстро только первый и последний элемент.
3. HashMap, HashSet — реализации на основе функции hashCode
4. TreeMap, TreeSet – элементы (ключи) должны быть Comporable, реализация на основе
дерева. Добавляют понятие упорядоченности. Лучшее время поиска, но большее
накладные расходы на построение чем на основе функции hashCode
